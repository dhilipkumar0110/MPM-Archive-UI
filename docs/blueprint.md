# **App Name**: ArchivaDash

## Core Features:

- Archival Dashboard: Interactive dashboard displaying key metrics like task execution counts, average runtimes, errors, and an overview of scheduled archival tasks in a navigable grid, similar to the provided reference image.
- Task Detail & Progress Monitoring: Dedicated screen for a selected archival task, showing detailed execution logs, status, and real-time progress as tasks run daily or on a custom schedule.
- Ruleset Configuration & Upload: Screen for users to upload archive rulesets via Excel files, which are then parsed, validated, and displayed as editable cards.
- Advanced Where Clause Builder: A graphical, interactive tool for users to construct complex SQL-like WHERE clauses with multiple conditions to precisely filter data for archival from source tables.
- AI-Assisted Query Generation: Utilize generative AI as a tool to assist users in building 'where clause' conditions by suggesting filters based on natural language input or database schema analysis.
- Scheduled Task Management: Functionality to define, manage, and view schedules (daily, weekly, or custom) for individual archival tasks.

## Style Guidelines:

- Primary color: A confident and professional medium blue (#2672DB), evoking reliability and clarity in data management.
- Background color: A very light, desaturated blue (#ECF3FF), providing a clean and expansive canvas for information display in a light scheme.
- Accent color: A soft, bright blue-green (#69CFE2), used sparingly to highlight key actions or states, ensuring contrast and visual interest.
- Body and headline font: 'Inter' (sans-serif) for its modern, neutral, and highly readable qualities, ideal for data-intensive displays.
- Clean, line-based, and universally recognized system icons for clear communication of actions, statuses, and data types across the dashboard and task views.
- Structured, responsive grid-based layouts with clear card components for metrics and interactive data tables for tasks, optimizing for both desktop and tablet experiences. Detailed screens will adopt a multi-column, organized flow.
- Subtle and functional animations for data updates, navigation transitions, and interactive elements within the where clause builder, providing a polished and responsive user experience without distraction.
